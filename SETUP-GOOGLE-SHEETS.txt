========================================================================
  CONNECT GOOGLE SHEETS  (personal Gmail / OAuth method)
  Auto-creates an editable sheet per client in YOUR Google Drive.
  One time, ~15 min. Free.
========================================================================

You already have the Google Cloud project with Sheets API + Drive API ON.
Now we get a one-time "refresh token" so the tool can act as you.

STEP 1 — OAuth consent screen
1. console.cloud.google.com -> APIs & Services -> OAuth consent screen.
2. User type: External -> Create.
3. App name: Oakline. User support email: your email. Developer email: your email. Save & continue.
4. Scopes: skip (Save & continue).
5. Test users: ADD your own Gmail (the one whose Drive will hold the sheets). Save.
6. Back on the OAuth screen, click "PUBLISH APP" -> Confirm.
   (Publishing keeps the token from expiring. drive.file is a low scope, so no
    Google review is needed; you may see an "unverified app" notice for your own
    account, which is fine to continue past.)

STEP 2 — Create an OAuth Client ID
1. APIs & Services -> Credentials -> + Create Credentials -> OAuth client ID.
2. Application type: Web application. Name: Oakline Web.
3. Under "Authorized redirect URIs" -> Add URI:
       https://developers.google.com/oauthplayground
4. Create. A popup shows your Client ID and Client secret -> copy BOTH to Notepad.

STEP 3 — Get the refresh token (OAuth Playground)
1. Go to  developers.google.com/oauthplayground
2. Top-right GEAR icon -> tick "Use your own OAuth credentials"
   -> paste your Client ID and Client secret -> close.
3. On the LEFT, find the box "Input your own scopes" and paste exactly:
       https://www.googleapis.com/auth/drive.file
   -> click "Authorize APIs".
4. Sign in with YOUR Gmail (the sheets owner). If it warns "unverified",
   click Advanced -> "Go to Oakline (unsafe)" -> Allow.
5. Back in the Playground, click "Exchange authorization code for tokens".
6. Copy the "Refresh token" value -> Notepad.

STEP 4 — Put the 3 values in Vercel
1. Vercel -> your project -> Settings -> Environment Variables. Add:
       GOOGLE_OAUTH_CLIENT_ID       = (Client ID)
       GOOGLE_OAUTH_CLIENT_SECRET   = (Client secret)
       GOOGLE_OAUTH_REFRESH_TOKEN   = (the refresh token)
2. Save -> Deployments -> Redeploy.

DONE. In the tool, after a hunt, click "Save to Google Sheet":
- It creates the sheet in YOUR Drive, shares it editable with the signed-in
  client's email, and opens the link.

If "Save to Google Sheet" errors, re-check the 3 values and that you clicked
"Exchange authorization code for tokens" (not just Authorize).
